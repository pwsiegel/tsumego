# Collaboration notes
The purpose of this project is to create an app that will help users extract go problems from PDFs, train with those tsumego, and interact with their go teacher regarding their tsumego training. This document will be updated if the scope of the app increases.

- You will typically be running in dangerous mode. As a result, please be diligent about consulting with me before making major changes, or especially irreversible changes. If I directly ask you to do something and it is safe, go ahead and do it.
- Clean, readable, well-structured code is important to me, and worth doing extra work.
- I generally want the ability to run things myself, and to that end I would like you to help me maintain the Makefile so that it includes common tasks like training and running dev servers.
- My budget for this project is very small, so when exploring cloud training or deployment options always try to understand what I am trying to achieve and recommend cheaper ways I can achieve my objectives (or slightly scale back my objectives if cost savings are significant).
- Maintain `lab-notebook.md` in the project root. Every time we try a new approach to a problem, add a short entry describing what it is, when we tried it, and what it's meant to achieve. When an approach fails or gets scrapped, update the same entry with the failure mode and the reason it didn't work — so future-us doesn't repropose it without remembering why it was rejected. Treat the notebook as a running record, not a planning doc.
- **Feedback-mode pacing**: after I deliver a change for you to try (especially UI / dev-tool changes), expect a stream of feedback from you — observations, screenshots, edge cases, partial thoughts — landing in several messages in quick succession. A substantive reply from me on the first message tends to collide with the next one you're already typing and trips us both up. Default to brief acknowledgements until you ask a direct question, give a directive, or otherwise signal you're done observing. Save analysis, plans, and tool calls for after that signal.
